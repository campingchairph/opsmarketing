📧 Marketing Tasks

Salesforce EDM — Sending a Designed Email

This is the most important task in the role. 16 steps across 7 phases. Read carefully — mistakes here reach thousands of brokers.

ℹ️
What is an EDM? EDM = Electronic Direct Mail. It's a professionally designed HTML email — like a digital newsletter — sent to Assetline's database of mortgage brokers. Petro designs it, you send it.
Phase 1 — Receive & Open Files
  • 1
    Receive the ZIP file from Petro via Teams

    Petro sends a ZIP file via Microsoft Teams. It includes: the HTML email file, campaign name, send date and time, and sometimes notes about the sender details.

    As soon as it arrives, open the Campaign Details card at the top of the Salesforce EDM page in the Marketing Hub app and fill in the campaign name and any sender changes Petro mentions. Do this now — not later.

  • 2
    Extract the ZIP and open both files in Chrome

    Inside the ZIP is a date-named folder. Inside that folder you'll find two files: index.html (the designed email) and plain-text-version.txt.

    Right-click each file → "Open with Google Chrome." Keep both Chrome tabs open the entire time. Don't close them until the send is complete.

Phase 2 — Create the Template in Salesforce
  • 3
    Create a new template in Account Engagement

    In Salesforce: go to Account Engagement → Email → Templates → New

    Name it using this format: YYMMDD_Description — e.g., 260701_HorizonMortgages

    Set Campaign to exactly match the Campaign Details card. Tracker domain must be go.assetline.com.au. Click Save.

  • 4
    Skip the layout step

    After saving, Salesforce shows a layout selector. Skip this entirely. The HTML file from Petro already has the complete email layout built in. Choosing a Salesforce layout on top of it will break things.

  • 5
    Build the email — subject line, HTML, and plain text

    Subject line: Look at the Chrome tab title showing index.html — that is your subject line. Type it exactly. Check every word.

    HTML tab: In Chrome on the email page, click anywhere in the email body → right-click → View Page Source → Ctrl+A → Ctrl+C. Go to the HTML tab in Salesforce, delete everything there, paste, and save.

    Text tab: Go to the other Chrome tab (plain text file) → Ctrl+A → Ctrl+C → paste into Salesforce's Text tab → save.

Phase 3 — Review & Fix
  • 6
    Check the email in the Editor tab

    Click the Editor tab — the email should display visually. Read the wording, check the layout, confirm the content is correct. Small spacing glitches in the editor are normal and won't appear in the actual sent email.

  • 7
    Check every link — especially LinkedIn

    Click every icon and button in the email and confirm each one has a working URL. The LinkedIn icon almost always arrives with no URL attached — this is a known issue with how the HTML is generated.

    Every single time, manually add: https://linkedin.com/company/assetline-capital

    🔴 Do this every send — never assume it's already there
  • 8
    Verify merge fields in the Preview tab

    Go to the Preview tab. Select a sample contact (e.g., "Billy"). Check that the first name field shows an actual name — not code like {{FirstName}}.

    This is the only place merge fields display correctly. They won't show in test emails. Then save.

    ⚠️ Only verifiable in Preview — not in test sends
Phase 4 — Sender Setup
  • 9
    Set sender name, email address, and reply-to

    Go to the Sending tab. Read the Campaign Details card — Petro specifies these for each campaign. They change every time.

    Set sender name → save. Sender email → save. Reply-to → save.

    🔴 Never copy from a previous send — always read from the card
Phase 5 — Test & Publish
  • 10
    Send a test to yourself — not Petro yet

    Send a test email to your own email address only. Check: layout looks correct, all links work, wording is right, no errors. Fix anything before Petro sees it.

  • 11
    Publish the template

    Once satisfied with your own test, click Publish from the Editor tab. This does NOT send to anyone. It simply makes the template available for use in a Send. No recipients receive anything at this point.

    💡 Publishing ≠ Sending. Safe to do.
Phase 6 — Build the Send
  • 12
    Create a new Send

    Go to the Sends tab → New Send. Name it the same as your template: YYMMDD_Description. Set the Campaign from the card. Save.

  • 13
    Apply the template — then notify Petro

    In the Send, search for your template by name and select it. Subject line, HTML, and sender details all carry over automatically.

    Now send a test to Petro. This is the first time he sees it. Use the Teams message from the notification widget on the Marketing Hub page.

  • 14
    Add all four recipient lists — every send, no exceptions

    You must add all four of these every single time:

    • ✅ All Brokers (Leads + Contacts)
    • ✅ James Green Database
    • ✅ Assetline Team Managerial/Executive
    • ✅ State Managers
    🔴 Missing a list = people who should get it, don't
  • 15
    Run the Final QA checklist

    The Marketing Hub has a QA checklist on the Salesforce EDM page. Go through every item and confirm before proceeding to the final step.

Phase 7 — Send or Schedule
  • 16
    Send now or schedule at the correct local time

    If Petro says send immediately → click "Send Now."

    If Petro gives a specific time → click "Schedule." Salesforce uses your Manila local time. If Petro says "9 AM Sydney" — subtract 3 hours and schedule at 6 AM Manila time.

    🔴 Wrong timezone = email goes at the wrong time
    💡
    The Marketing Hub header shows Manila and Sydney clocks side by side — use it to calculate the correct time.

🧠 Quick Check

You're about to send an EDM. You've added 3 of the 4 lists — All Brokers, James Green, and State Managers. Should you send?
Yes — 3 out of 4 is close enough for now
No — all 4 lists are required, every time
Only if Petro specifically asked for the 4th list

Plain Text Emails

Simpler than an EDM — no design, just words. These appear to come directly from a person like Laura Stanley.

ℹ️
What's different about plain text emails? No HTML design, no images, no layout blocks. It reads like a personal email from a colleague. They go through Email Content (not Account Engagement Email) in Salesforce. You prepare it — the sender (e.g., Laura) activates and sends it herself.
⚠️
Plain text body = internal only. The customer never sees the plain text version. It exists so Salesforce contact records show what was sent without loading the full HTML. Always sync it — but it's not customer-facing.
The 5-Step Workflow
  • 1
    Clone the previous email

    In Salesforce Account Engagement: Content → Email Content. Find the most recent email for the same product (e.g., Private Lending). Click the dropdown arrow → Clone → enter the new name using the naming convention → Save.

    Cloning keeps the template structure so you're not starting from scratch — it's faster and less error-prone.

  • 2
    Update the subject line

    Open the cloned email → Edit → New Email Experience (already selected — just click through). Click the edit icon on the subject line. Copy the subject from the Word document provided → paste → save.

  • 3
    Update the body copy only — nothing else

    Edit only the first block — the body copy. Everything below (sender name, title, email, phone number) is controlled by merge fields. Do not touch those.

    Delete the old body text → paste the new body text from the Word document → fix any spacing → save.

    ⚠️ Check if "chat soon" is already in the template — don't duplicate it
  • 4
    Sync the plain text body

    In the email view, find the Text Body section → click the pencil icon → click Sync from HTML → Confirm → Save. This updates the internal Salesforce record to match the email body.

  • 5
    Notify the sender — you don't send it yourself

    Copy the link to the email record in Salesforce and send it to Laura (or whoever the sender is) with a message saying it's ready. She handles the final activation and send. Your job is done at this point.

    ✓ Your role ends at "it's ready"

InDesign Essentials

Adobe InDesign is used to lay out print and digital documents — brochures, rate cards, invitations, event collateral. You don't need to design from scratch, but you need to know how to edit and QC files correctly.

✅
Your role in InDesign You're working with existing Assetline templates — not creating designs from scratch. Your job is to: place approved copy into the layout, check everything looks right, fix issues, and export a clean PDF for Petro's approval.
Opening & Setting Up a File
📂

The right way to open an InDesign file

Don't skip these steps
▼
  • 1
    Always open the packaged folder — not a loose file

    A properly packaged InDesign job is a folder containing the .indd file, a Links folder (images), and a Document fonts folder. Always open the .indd from inside this folder — never move the files around.

  • 2
    Check the Links panel immediately

    Window → Links. If any image shows a yellow warning triangle (modified) or a red circle (missing), fix it before doing anything else. Click the warning → Relink → navigate to the correct file.

  • 3
    Check for font substitutions

    Type → Find Font. If InDesign is substituting a font it can't find, it shows up here with a warning icon. Install the missing font or relink to the correct one. Pink/highlighted text in the document = substituted font.

Editing Text in InDesign
✏️

How to update copy correctly

Paragraph styles are everything
▼
  • 1
    Use the Type tool (T) to click into a text frame

    Press T to activate the Type tool. Click inside a text frame to start editing. Double-click a frame to enter it directly.

  • 2
    Delete old text and paste new text — then apply the correct paragraph style

    Select the old text → delete. Paste the new text. Then open the Paragraph Styles panel (Window → Styles → Paragraph Styles). Click the correct style name to apply it. Never manually change font, size, or colour — always use the style.

    ⚠️ A + symbol next to a style name means manual overrides exist — click the style name while holding Alt/Option to clear them
  • 3
    Check for overset text after every edit

    A red + in the bottom-right of a text frame means text is cut off (overset). Either reduce the text length, increase the frame size, or reduce the font size — but only if the style allows it. Flag to Petro if you're unsure.

⚠️
Never use free transform to resize a text frame if it distorts content. Hold Shift when resizing images to maintain proportions. Text frames can be freely resized — images cannot be distorted.
Exporting a PDF from InDesign
📤

File → Export — the right settings

Getting the PDF right first time
▼
  • 1
    Run Preflight first — zero errors required

    Window → Output → Preflight. The panel at the bottom of the screen shows a green circle (no errors) or red circle (errors found). Fix all errors before exporting. Common errors: missing images, overset text, wrong colour space.

  • 2
    Switch to Preview mode to check appearance

    View → Screen Mode → Preview (or press W). This hides all guides, margins, and bleed indicators so you see exactly what the output will look like. Check every page.

  • 3
    Export: File → Export → Adobe PDF (Print)

    For print jobs: Choose PDF/X-1a or PDF/X-4 preset. Set bleed to 3mm on all sides. Confirm "Use Document Bleed Settings" is checked.
    For digital/screen: Choose "High Quality Print" preset. No bleed needed.

  • 4
    Open the PDF in Acrobat and scroll every page

    Never assume the export was clean. Open the PDF, go full screen, and scroll through every page. Look for: cut-off text, missing images, wrong colours, blank pages, incorrect page order. Only then file it and send to Petro.

PDF Editing & Acrobat

You'll use Adobe Acrobat to review PDFs, annotate for corrections, compare versions, and make minor text edits. It's not for major design changes — that always goes back to InDesign or Illustrator.

Reviewing a PDF
🔍

How to review a PDF properly

Never just scroll past it quickly
▼
  • 1
    Open in Acrobat (not just a browser PDF viewer)

    Right-click the PDF → Open with → Adobe Acrobat. A browser preview won't show all issues — Acrobat will.

  • 2
    View at 100% zoom and scroll every single page

    Set zoom to 100% (Ctrl+1). Go page by page. Don't skim. Look for: cut-off text, missing images, wrong fonts, broken layout, incorrect colours, wrong page order, blank pages.

  • 3
    Check the disclaimer and legal text

    These are usually small. Zoom in (Ctrl+= to zoom in) and read them. Check the AFSL number is present and the general advice warning is verbatim.

When you find an issue in a PDF, don't try to fix it in Acrobat (unless it's a tiny text fix). Instead, annotate the PDF with a comment and send it back to be fixed in the original InDesign/Illustrator file.

Minor Text Edits in Acrobat
✏️

When you can (and can't) edit directly in Acrobat

Only for small, approved fixes
▼
⚠️
Only edit in Acrobat for very minor fixes — a wrong date, a typo in one word, a wrong phone number. Never use Acrobat for layout changes, font changes, image swaps, or anything that changes the design. Those must go back to InDesign.

To edit text in Acrobat: Tools → Edit PDF → Edit. Click the text you want to change. Acrobat loads the editable text block. Make your change. Be careful — Acrobat may not have the exact font installed and may substitute it.

Text Compare — Your QC Weapon

The Marketing Hub has a Text Compare tool built in. Use it to verify copy exactly matches the approved brief — word for word, character for character.

✅
Why Text Compare matters for QC When you place copy from a Word document into InDesign or an email template, it's easy to accidentally change a word, miss a line, or introduce a spacing error. Text Compare catches these without you having to read both versions manually side by side.
Three Compare Modes
📄

Mode 1: Text Compare

Paste two versions of text — highlights every difference
▼

Where: Marketing Hub → Reference → Text Compare → Text tab

  • 1
    Paste the approved copy on the left

    Copy the approved text from the brief document (Word file or email from Petro) and paste it into the left panel.

  • 2
    Paste your placed copy on the right

    Copy the text from your InDesign file, email template, or wherever you placed the copy, and paste into the right panel.

  • 3
    Read the highlighted differences

    The tool highlights every line that differs. Changed words are highlighted within the line. If there are zero differences — you're clean. If there are differences — go back and fix the placed copy to match the brief.

💡
Use this to compare: email copy vs brief, rate card numbers vs approved data, disclaimer text vs the standard wording, or two versions of the same document.
📁

Mode 2: Doc / PDF Compare

Upload two Word or PDF files — no manual copy-paste needed
▼

Where: Marketing Hub → Reference → Text Compare → Doc/PDF tab

Upload two versions of a Word (.docx) or PDF file. The tool extracts the text from both files automatically and runs the same line-by-line diff. No manual copying needed.

When to use this:

  • Comparing two versions of a rate card
  • Checking an updated T&C document against the previous version
  • Verifying a revised brief against the original
✅
All file processing happens in your browser — nothing is uploaded to a server. The content stays private.
🖼️

Mode 3: Image Compare

Spot visual differences between two artwork versions
▼

Where: Marketing Hub → Reference → Text Compare → Image tab

Upload two image files (e.g., before and after artwork, two versions of a banner or social graphic). The tool gives you three views:

ViewWhat it shows
Side by sideBoth images next to each other at the same zoom level
Swipe comparisonDrag a divider left and right to reveal one image over the other — great for spotting layout shifts
Pixel difference overlayHighlights exactly which pixels differ between the two images — useful for catching tiny changes
💡
Use the swipe view when checking if a revised social graphic has moved any elements compared to the approved version. The pixel diff view is good for catching colour changes or subtle text edits.
