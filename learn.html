Your role in plain English

You are the Marketing Coordinator at Assetline Capital — a financial company that lends money to property developers and investors.

📧 Stage 3

Sending an EDM Email

This is the most important task in the role. An "EDM" is a designed marketing email sent to all brokers via Salesforce. There are 16 steps — we'll go through each one.

⏱ About 90 minutes
ℹ️
What is an EDM? EDM stands for "Electronic Direct Mail." It's a nicely designed HTML email — like a digital newsletter or product announcement — sent to Assetline's database of mortgage brokers.
Phase 1 — Receive & Open Files
  • 1
    Receive the ZIP file from Petro via Teams

    Petro sends a Microsoft Teams message with a ZIP file attached. The message will include: the HTML email file, campaign name, send date and time, and sometimes special notes about the sender details.

    As soon as you receive it, update the "Campaign Details" card at the top of the Salesforce EDM page in the Marketing Hub app with the campaign name and any sender changes Petro mentions.

  • 2
    Extract the ZIP and open both files in Chrome

    Inside the ZIP folder, there will be a date-named folder. Inside that, find two files: index.html (the designed email) and plain-text-version.txt.

    Right-click each file → "Open with Google Chrome." Keep both Chrome tabs open the entire time you're working on this send.

Phase 2 — Create the Template in Salesforce
  • 3
    Create a new email template in Salesforce

    In Salesforce Account Engagement, go to: Email → Templates → New

    Name the template using this format: YYMMDD_Description (e.g., 260630_HorizonMortgages_NewRates)

    Set the Campaign to match exactly what's in your Campaign Details card. Tracker domain must be go.assetline.com.au. Then click Save.

  • 4
    Skip the layout step

    On the screen after saving, Salesforce shows layout options. Skip this step entirely.

    The HTML file from Petro already includes the full email layout — you don't need Salesforce to add one.

  • 5
    Build the email: subject line, HTML content, plain text

    Subject line: Look at the Chrome tab title for index.html — that's your subject line. Type it in exactly. Double check the spelling.

    HTML content: In your Chrome tab with the email open, click inside the email body → then right-click on the page → "View Page Source" → Select All → Copy. Then in Salesforce's HTML tab, delete whatever's there and paste. Save.

    Plain text: Go to your other Chrome tab (the plain text version) → Select All → Copy → paste into Salesforce's Text tab. Save.

Phase 3 — Review & Fix
  • 6
    Check everything in the Editor tab

    Go to the Editor tab in Salesforce — the email should show up visually. Read the wording, check the content, make sure the layout looks right.

    Don't panic if spacing looks slightly off in the editor — this is normal. The actual sent email will look correct.

  • 7
    Add missing links — especially LinkedIn

    Check every clickable icon (social media icons, buttons, logos) for missing URLs. The LinkedIn icon almost always comes through with no link attached.

    Manually add the Assetline LinkedIn URL every single time: linkedin.com/company/assetline-capital

    🔴 Critical — do this every send, every time
  • 8
    Verify merge fields in the Preview tab

    Go to the Preview tab and select a sample contact (e.g., "Billy"). Check that the first name field shows the actual name, not a code like {{FirstName}}.

    This is the only way to verify merge fields — they won't show up in test emails. Then save.

    ⚠️ Merge fields only work in Preview — not in test sends
Phase 4 — Sender Setup
  • 9
    Set the sender name, email, and reply-to

    Go to the Sending tab. Look at the Campaign Details card at the top of the Marketing Hub page — Petro specifies these for each send.

    Set: Sender name → Save. Sender email → Save. Reply-to → Save.

    These change campaign to campaign. Always read from the card. Never assume they're the same as last time.

    🔴 Critical — wrong sender = serious problem
Phase 5 — Test & Publish
  • 10
    Send a test email to yourself first

    Before anything goes to Petro, send a test to yourself only. Check: does the layout look right? Do all the links work? Is the wording correct? Does your name appear where the merge field is?

  • 11
    Publish the template

    Once you're happy with your own test, click "Publish" from the Editor tab. This does NOT send the email to anyone — it just makes the template available to use in a Send (next phase).

    💡 Publishing ≠ Sending. No one receives anything yet.
Phase 6 — Build the Send
  • 12
    Create a new Send in the Sends tab

    Go to the Sends tab → New Send. Name it the same as your template: YYMMDD_Description. Set the Campaign from the Campaign Details card. Save.

  • 13
    Apply the template, then notify Petro

    In the Send, search for your template by name and select it. All the subject line, HTML, and sender details carry over automatically.

    Now send a test to Petro. This is the first time he sees it. Use the Teams message from the notification widget on the Marketing Hub page to let him know.

  • 14
    Add ALL four recipient lists — every single time

    This step is critical. Add all four of these lists to every send, no exceptions:

    • ✅ All Brokers (Leads + Contacts)
    • ✅ James Green Database
    • ✅ Assetline Team Managerial/Executive
    • ✅ State Managers
    🔴 Critical — missing a list = people who should get it, don't
  • 15
    Run the Final QA checklist on the page

    The Marketing Hub has a QA checklist on the Salesforce EDM page. Go through every item. All must be confirmed before you proceed to the final step.

Phase 7 — Send or Schedule
  • 16
    Send now or schedule at the right time

    If Petro says send immediately → click "Send Now."

    If Petro gives a specific time → click "Schedule." Important: Salesforce uses your local Philippines time.

    If Petro says "9 AM Sydney time" — subtract 3 hours. Schedule it at 6 AM your time.

    🔴 Wrong timezone = email goes at the wrong time
    💡
    The dual clock in the Marketing Hub header shows Sydney and Manila time side by side — use it to calculate the right local time to schedule.

🧠 Quick Check — Stage 3

Yes — three lists is enough, State Managers and All Brokers overlap anyway
No — you must add all 4 lists every single time without exception
Only if Petro specifically mentioned the 4th list in his instructions

✉️ Stage 4

Plain Text Emails

A different type of email — simpler, no design, sent as if it came directly from a person like Laura Stanley.

⏱ About 30 minutes
ℹ️
Plain text emails are different from EDMs. There's no HTML design, no images, no layout. It reads like a personal email from a colleague. These go to brokers and look like Laura Stanley wrote them herself.
The 5-Step Plain Text Workflow
  • 1
    Clone the previous email

    In Salesforce Account Engagement, go to: Content → Email Content. Find the most recent email for the same product. Click the dropdown arrow next to it → Clone → Enter the new name → Save.

    Cloning saves time — you're starting from the right template structure instead of from scratch.

  • 2
    Update the subject line

    Open the cloned email → Edit → New Email Experience (it's pre-selected, just click through). Click the edit icon on the subject line field. Copy the subject from the Word document provided by Petro or Laura → paste → Save.

  • 3
    Update the email body — body only

    Edit only the first block — the body copy. Everything below (sender name, title, email, phone) uses merge fields — do not touch these. Delete the old body copy, paste the new body copy from the Word document, fix any spacing, then Save.

    ⚠️ Don't duplicate the closing line — check if "chat soon" is already in the template
  • 4
    Sync the plain text body

    In the email view, find the Text Body section → click the pencil icon → click "Sync from HTML" → Confirm → Save.

    This updates the internal Salesforce record to match the email body. The customer never sees the plain text body — it's just for internal CRM records so you can see what was sent.

  • 5
    Notify the sender — you don't send it yourself

    Copy the link to the email record and let Laura (or whoever the sender is) know it's ready. You do not send this email. The sender handles the final activation and send themselves.

    💡 Your job ends at "it's ready" — the sender pushes the button

Brand & Quality Control

Before anything goes to Petro, you run through this checklist. Brand and compliance mistakes can be costly — this is how you catch them.

⏱ About 45 minutes

Why QC matters so much 🔍

Assetline Capital is a financial company. Wrong rates, missing disclaimers, or wrong logos aren't just embarrassing — they can be legal and compliance issues. Always QC before submitting.

📊 Stage 6

EDM Reporting & Brief Intake

What happens after an email is sent — and how to capture new tasks properly.

⏱ About 30 minutes
EDM Reporting
📈

Building a campaign performance report

What to measure after each send
▼

After each email campaign goes out, Petro may ask for a performance report. You build this in the EDM Reporting tab of the Marketing Hub.

What you collect from Salesforce for each campaign:

  • Campaign name
  • Send date
  • Subject line
  • Opens (how many people opened it)
  • Clicks (how many people clicked a link)
  • Unsubscribes (people who opted out)
  • Bounces (emails that couldn't be delivered)
  • Click map image (screenshot of where people clicked)
💡
The reporting page saves all your entries automatically. You can export the report as a formatted view to share with Petro. Data persists between sessions so you can build it up over time.
Brief Intake
📋

Capturing new tasks properly

Do this before starting any task
▼

Every time you receive a new task — from Petro or anyone else — fill in the Brief Intake form in the Marketing Hub (Brand Tools → Brief Intake) before you start working on it.

What you record:

  • Task type and description
  • Who assigned it
  • Deadline
  • Deliverable format (PDF? HTML? Word doc?)
  • Special notes or constraints
⚠️
Fill this in BEFORE you start — not after. It takes 2 minutes and prevents the "I didn't realise that was needed" problem when you're already halfway through.

Past briefs are saved and you can review them directly on the page. This becomes your work history.
